DMP

What is DMP?

Data Memory-Dependent Prefetcher (DMP) is a hardware component designed to optimize memory access performance. It predicts the memory addresses that the program will access and preloads the data, thereby reducing memory access latency. Traditional prefetchers base their predictions on historical access patterns, whereas DMP uses the contents of data memory to make predictions, handling irregular memory access patterns.

How DMP Works?

Step1: Observing Cache Line Data

The DMP scans the data in an L1 line when that line is filled to the L1, if the line is not marked with the “do-not-scan” hint (i.e., the line has not been scanned since it was brought into the cache). The DMP performs the scan by checking each pointer size-aligned chunk (the first 64 bits, second 64 bits, etc.) in the cache line

Step2: Address Check

Next, the DMP applies additional checks and filters to each chunk (candidate pointer) to see if it should be dereferenced. If the value represents an address within the 4GB vicinity of the current address, DMP will consider this value as an address and perform preloading.
